# Release and publication checklist

## Release stages

### Stage A - public working release

Ready before the prospective campaign closes:

- empirical paper v0.1 clearly marked as historical manuscript;
- methods paper v0.9 clearly marked as protocol paper;
- MARB Beta contract, schema, validator and catalog;
- synthetic contract-test result;
- external review packet and adversarial review form;
- portfolio and competition packages;
- repository commit and artifact hashes.

This stage supports public sharing, professor outreach, portfolio use and methods feedback. It does not support a final prospective-transfer claim.

### Stage B - campaign-closed release

After the frozen campaign ends:

- run all final evaluators without changing gates;
- preserve failed, inconclusive or insufficient-volume outcomes;
- update only the reserved empirical-paper section, abstract wording and discussion;
- update benchmark prospective status and evidence tier only when gates permit;
- regenerate PDFs and evidence audits;
- tag the release and record checksums.

### Stage C - formal submission release

Before a journal, workshop or conference submission:

- complete at least one external adversarial review;
- resolve S3/S4 findings or narrow the claim;
- verify data redistribution and citation rights;
- replace author placeholders and add affiliations;
- select a venue-specific template;
- freeze a submission commit;
- archive code, paper PDFs, claim registries and reproduction commands;
- create a versioned DOI or repository archive when available.

## Release naming

Recommended tags:

- `research-assets-v0.2` - historical papers and benchmark Beta;
- `prospective-close-v1.0` - final frozen seven-day result;
- `submission-<venue>-v1` - exact formal-submission snapshot.

## Artifact set

Each release should retain:

- empirical paper PDF and Markdown;
- methods paper PDF and Markdown;
- claim registries and audit reports;
- benchmark specification, schema, catalog and reference submission;
- source commit SHA;
- model and workflow artifact digests;
- prospective evaluation output when mature;
- exact environment and reproduction commands.

## Non-negotiable publication boundaries

Do not claim stable profit, live readiness, scalable capacity, validated home-only strategy or cross-domain empirical transfer without new evidence. A positive prospective mechanism result changes only the transfer tier. Operational claims require separate execution, account, risk and capacity evidence.
